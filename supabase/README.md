# Supabase Local Environment

M1 initializes the Supabase local project with `npx supabase init`.

## Commands

```powershell
npx supabase --version
npx supabase start
npx supabase status
npx supabase stop
```

Schema migrations, RLS policies, storage policies, seed data, and Edge Functions are intentionally deferred to M2 or later approved phases.

## Notes

- `supabase/config.toml` is generated by the Supabase CLI.
- `supabase/seed.sql` is intentionally empty in M1 because canonical seed data belongs to M2.
- Use environment variables or Supabase secret management for secrets; never commit service keys.
